## 二、术语详解

### 1. Node.js
- **是什么**: JavaScript 的服务器端运行环境
- **类比**: 就像 Python 解释器让 .py 文件能运行，Node.js 让 .js 文件能在服务器上运行
- **你的项目**: 用于运行 `npm` 命令、Vite 构建工具等

### 2. React
- **是什么**: 用于构建用户界面的 JavaScript 库
- **运行位置**: 浏览器中
- **作用**: 把 UI 拆分成可复用的组件
- **你的项目**: ✅ 使用中 - `App.tsx`、`ExpenseForm.tsx` 等都是 React 组件

### 3. Vite
- **是什么**: 现代前端构建工具
- **运行位置**: 开发时在 Node.js 环境运行
- **作用**:
  - `npm run dev` - 启动开发服务器，热更新
  - `npm run build` - 打包成静态文件
- **你的项目**: ✅ 使用中

### 4. Next.js
- **是什么**: 基于 React 的全栈框架
- **运行位置**: 服务器 + 浏览器
- **作用**: 提供 SSR（服务端渲染）、API 路由等
- **你的项目**: ❌ 未使用

### 5. TypeScript
- **是什么**: JavaScript 的超集，添加了类型系统
- **运行位置**: 编译时在 Node.js 运行，编译后变成 JS 在浏览器运行
- **你的项目**: ✅ 使用中

## 三、前端 vs 后端运行位置

| 技术 | 运行位置 | 说明 |
|------|----------|------|
| React 组件代码 | 浏览器 | 用户访问网页时下载并执行 |
| Vite 构建工具 | 开发机器 | 只在开发时使用 |
| Express.js API | 服务器 | 处理数据库、认证等 |
| Next.js API Routes | 服务器 | 类似 Express，但集成在 Next.js 中 |

## 五、Netlify 部署的本质

Netlify 托管的是**静态文件**:
- 没有 Node.js 服务器在运行
- 所有代码都在用户浏览器中执行
- 数据存储在用户浏览器的 localStorage 中

## 六、关于密钥安全

### ⚠️ 重要警告

**前端代码中放置密钥是不安全的！**

原因:
1. Vite 的 `define` 会将环境变量值**硬编码**到打包后的 JS 文件中
2. 任何人打开浏览器开发者工具都能看到
3. 即使 `.env.local` 不提交到 Git，构建后的文件仍然包含密钥

### 你的项目当前状态

- `vite.config.ts` 配置了 `GEMINI_API_KEY` 的注入
- 但代码中目前没有使用这个密钥
- 如果将来使用，密钥会暴露

### 解决方案

如果需要使用 API 密钥，有以下选择:

1. **使用后端代理** (推荐)
   - 部署一个简单的 serverless function
   - Netlify Functions 支持这个功能

2. **使用 Netlify 环境变量 + Functions**
   - 密钥存在 Netlify 的环境变量中
   - 通过 Function 在服务器端调用 API
   - 前端只调用你自己的 Function

3. **如果是公开 API**
   - 某些 API 设计为可在前端使用
   - 检查 API 文档是否有使用限制和最佳实践

## 七、常用命令速查

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 检查 Node.js 版本
node -v

# 检查 npm 版本
npm -v
```
