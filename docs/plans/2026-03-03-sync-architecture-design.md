# 同步架构重构设计文档

## 背景

用户报告了以下问题：
1. 删除账单操作没有触发远程删除，同时两个人打开时，删除的账单又回来了
2. 同一账户登录，在一个浏览器上创建同步过的账本，在另一个设备上没有自动下载
3. 需要改进本地缓存策略，以云端为准
4. 站点进入逻辑和账本创建流程缺乏仪式感

## 需求总结

| 问题 | 解决方案 |
|------|----------|
| **删除同步** | 乐观更新 + 冲突解决（用 `deletedAt` 标记已删除的账单） |
| **账本列表同步** | 合并模式 - 登录后从云端拉取并合并到本地 |
| **本地缓存** | 保留本地缓存，但以云端为准，游客模式关闭后消失 |
| **账本创建引导** | 名称 → 目的地/出发地 → 货币/汇率 → 添加家庭成员 |
| **登录时机** | 可选登录 - 弹出提示，但可选择"游客模式体验" |

## 数据模型变更

### 账单 (Expense)

```typescript
interface Expense {
  id: string;
  ledgerId: string;
  date: string;
  description: string;
  amount: number;
  category: string;
  payerId: string;
  sharedWithFamilyIds: string[];
  createdBy: string;
  createdByDisplayName: string;

  // 新增字段
  version: number;           // 版本号，每次修改 +1
  deletedAt?: number;        // 软删除时间戳，存在则表示已删除
  updatedAt: number;         // 最后更新时间戳
}
```

### 账本 (LedgerMeta)

```typescript
interface LedgerMeta {

  ownerId?: string;

  status: 'active' | 'archived';
```

### 同步规则

- 删除操作：设置 `deletedAt` 而非物理删除
- 更新操作：`version++`，更新 `updatedAt`
- 同步时：过滤 `deletedAt` 存在的记录，版本高的覆盖版本低的

## SyncService 核心架构

### 模块职责

- **VersionManager**: 版本号管理，每次 CRUD 操作自动 +1
- **ConflictResolver**: 冲突解决，版本高者胜；版本相同则 updatedAt 决定
- **OfflineQueue**: 离线操作队列 (IndexedDB)，离线时存储操作，恢复在线后批量执行
- **LedgerSync**: 账本列表同步，登录后拉取用户所有账本

### 冲突解决策略

```typescript
function resolveConflict(local: Expense, remote: Expense): Expense {
  // 1. 版本号比较
  if (remote.version > local.version) {
    return remote;  // 远程版本更高，采用远程
  }
  if (local.version > remote.version) {
    return local;   // 本地版本更高，保留本地
  }

  // 2. 版本相同，比较更新时间
  if (remote.updatedAt > local.updatedAt) {
    return remote;
  }
  return local;
}
```

### 删除的特殊处理

```typescript
// 删除操作：设置 deletedAt，版本 +1
async function softDeleteExpense(ledgerId: string, expenseId: string) {
  const expense = await getExpense(ledgerId, expenseId);
  await updateExpense(ledgerId, expenseId, {
    deletedAt: Date.now(),
    version: expense.version + 1,
    updatedAt: Date.now()
  });
}

// 同步时过滤已删除的账单
function filterDeletedExpenses(expenses: Expense[]): Expense[] {
  return expenses.filter(e => !e.deletedAt);
}
```

## 错误处理

### 离线场景

- 操作存入 IndexedDB 离线队列
- 网络恢复后按时间顺序执行队列
- 先拉取云端最新数据，再应用本地更改

### 冲突场景处理

| 场景 | 处理方式 |
|------|----------|
| 双方同时编辑同一账单 | 版本号高者胜；相同则时间戳决定 |
| 一方删除，一方编辑 | 删除优先（`deletedAt` 存在则标记为已删除） |
| 离线多次修改 | 合并为一次更新，最终状态为准 |
| 网络恢复时冲突 | 先拉取云端，再应用本地更改，解决冲突后推送 |
